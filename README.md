# tanren

**T**echnical **A**gent for **N**urturing & **R**einforcing **E**ngineering **N**avigation

エンジニアの成長を支える壁打ち + マルチエージェント実力解析 CLI。

AI コーチと壁打ち（対話）を重ね、その履歴を軸ごとの専門エージェントが解析して、強み・弱み・次のアクション・到達度スコアをレポートします。続ける（壁打ち）→ 振り返る（解析）→ 次に取り組む、のループを回すためのツールです。

## インストール

```sh
npm install -g tanren
```

Node.js 18 以上が必要です。

## はじめに

```sh
tanren setup
```

AI プロバイダーを選びます（初回は `ask` / `report` 実行時に自動で案内されます）。

- **Claude** — Claude Code のログインを使用（API キー不要）
- **Gemini** — Google AI Studio の API キー（無料枠あり）

## 使い方

```sh
tanren ask       # AI コーチと壁打ちする
tanren report    # 壁打ち履歴から実力を軸ごとに解析する
tanren history   # 過去の解析レポートとスコア推移を見る
tanren history 2 # レポート #2 の詳細を見る
tanren actions   # 次に取り組むべきことを表示する
```

引数なしで `tanren` を実行すると、メニューから選べます。

## 仕組み

- **ask** — コーチは解析の軸を意識して問い返し、判断や設計の根拠を言語化させます。
- **report** — 軸ごとに専門エージェントが履歴を解析し、強み・弱み・次のアクションと 5 段階スコアを出します。前回レポート以降の新しい壁打ちだけを対象にし、前回の次アクションへの進捗も評価します。
- **history** — 軸ごとのスコアと推移、前回比を表示します。
- **actions** — 最新レポートの次アクションを軸ごとに一覧します。

## 解析する軸のカスタマイズ

評価する能力軸は `~/.tanren/axes.yaml` で編集できます。`label`（表示名）と `focus`（その軸で何を見るか）を書き換えると、次の `ask` と `report` から反映されます。`key` は過去レポートとの照合に使うため、既存の軸はそのまま残すのが無難です。

## データの保存先

すべて `~/.tanren/` 配下の YAML に保存されます（`sessions` / `reports` / `axes` / `config`）。外部に送信されるのは選んだ AI プロバイダーへの解析リクエストのみです。
